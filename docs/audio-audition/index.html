Replacement Workflow

  1. 在每个 cue 上点击当前占位音预听。
  2. 用 Upload Candidate 选择你找到的新音频，直接在网页里对比试听。
  3. 标记 Approved / Candidate / Replace / Reject，并写备注。
  4. 如果决定替换，把文件改名为目标文件名，覆盖到 Content/Audio/Placeholder/。
  5. UE 里运行 Content/Python/import_audio.py 和 bind_audio_cues.py。

网页不会直接写入项目文件。它只负责试听和记录决策，避免浏览器权限问题。
